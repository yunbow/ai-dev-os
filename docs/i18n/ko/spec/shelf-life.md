# AI 코딩 규칙의 수명 모델

## 개념

각 계층에는 "수명"이 있습니다 — 내용이 대대적인 수정이 필요하기까지의 예상 기간입니다.

| 계층 | 수명 | 비유 |
|------|------|------|
| L1: 철학 | 2-5년 | 도시 도로 배치 |
| L2: 판단 기준 | 1-3년 | 건물 건축 |
| L3: 가이드라인 | 6-12개월 | 인테리어 디자인 |
| L4: AI 프레임 | 2-4개월 | 가구 배치 |

## 투자 전략

상위 계층에 가장 많이 투자하세요. 오류가 하위로 복합적으로 전파되기 때문입니다:

- **L1 오류** (잘못된 철학) → 치명적, 거의 보이지 않음, 모든 것에 영향
- **L2 오류** (잘못된 원칙) → 비용이 높음, 감지하기 어려움, 모든 코드에 영향
- **L3 오류** (잘못된 가이드라인) → 보통, 리뷰로 감지 가능, 특정 패턴에 영향
- **L4 오류** (잘못된 도구 설정) → 저비용, 즉시 확인 가능, 쉽게 수정

## 만료 감지

`git log`를 사용하여 노후화를 감지합니다:

```bash
# 각 계층의 마지막 업데이트 날짜 확인
git log -1 --format="%ci" -- 01_philosophy/
git log -1 --format="%ci" -- 02_decision-criteria/
git log -1 --format="%ci" -- 03_guidelines/
git log -1 --format="%ci" -- 04_ai-frames/
```

`/ai-dev-os-audit` 명령이 이 검사를 자동화합니다.
