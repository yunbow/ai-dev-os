# AI 코딩 규칙의 의존성 규칙

## 원칙

> 상위 계층은 하위 계층을 참조해서는 안 됩니다.

이는 Robert C. Martin의 Clean Architecture 의존성 규칙과 Christopher Alexander의 패턴 언어 계층 구조에서 직접 차용한 것입니다.

## 계층별 규칙

### L1: 철학

- 포함해서는 안 되는 것: 도구 이름, 프레임워크 이름, 라이브러리 이름, API 이름
- 포함해서는 안 되는 것: 특정 언어의 코드 예제
- 포함할 수 있는 것: 추상적 개념, 가치, 사고 프레임워크

### L2: 판단 기준

- 포함해서는 안 되는 것: 프레임워크별 구현 세부사항
- 포함해서는 안 되는 것: 특정 라이브러리 API 참조
- 포함할 수 있는 것: 언어 패러다임 (OOP, FP), 일반 패턴 (MVC, CQRS)

### L3: 가이드라인

- 포함해서는 안 되는 것: 도구별 설정 (CLAUDE.md 형식, .cursorrules 구문)
- 포함할 수 있는 것: 프레임워크 이름, 라이브러리 이름, 코드 예제

### L4: AI 프레임

- 포함할 수 있는 것: 도구별 모든 것
- 참조해야 하는 것: L3 가이드라인을 원천 자료로

## 위반 탐지

`/ai-dev-os-audit` 명령은 다음을 스캔하여 의존성 위반을 검사합니다:

- L1 파일에서 도구/프레임워크 용어
- L2 파일에서 구현 세부사항

플러그인 훅도 L1-L2 파일을 편집할 때 실시간 경고를 제공합니다.
