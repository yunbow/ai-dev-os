# AI Dev OS へのコントリビューション

AI Dev OS への貢献に興味をお持ちいただきありがとうございます！

## コントリビューション方法

### Issue の報告
- [GitHub Issues](https://github.com/yunbow/ai-dev-os/issues) でバグ報告や改善提案を行ってください
- ドキュメントの問題を報告する際は、ファイルパスとセクションを含めてください

### Pull Request
1. リポジトリをフォーク
2. フィーチャーブランチを作成（`git checkout -b improve-spec`）
3. 変更を実施
4. 説明的なコミットメッセージでコミット
5. フォークにプッシュし、Pull Request を作成

### コントリビューション対象

| ディレクトリ | 求めている内容 | ガイドライン |
|-------------|--------------|------------|
| `spec/` | 修正、明確化、新しい設計原則 | エコシステム全体に影響するため、先に Issue で議論してください |
| `theory/` | 新しい理論的接続、研究参照の更新 | 事実に基づき、適切に引用してください |
| `getting-started/` | より良い例、新しい言語/ツールの組み合わせ | rules/plugin リポジトリと同期を保つ必要があります |
| `docs/i18n/` | 翻訳、翻訳の改善 | 下記の翻訳ガイドを参照 |

### 翻訳ガイド

- `spec/`, `theory/`, `getting-started/` の英語ファイルが Source of Truth
- 翻訳は `docs/i18n/{lang}/` に英語と同じ構造で配置
- 対応言語: `ja`（日本語）, `zh-CN`（中国語）, `ko`（韓国語）, `es`（スペイン語）
- 技術用語、コードブロック、URL、ファイルパスは英語のまま維持
- 各ファイル末尾に `Languages:` フッターで英語版へのリンクを追加

### バージョニング

| 変更種別 | バージョン |
|---------|----------|
| Spec の変更（4層モデル、依存性ルール） | MAJOR |
| 新しい theory ドキュメント、getting-started の更新 | MINOR |
| 誤字修正、表現の改善 | PATCH |

## 行動規範

敬意を持ち、建設的で、包括的であること。あらゆる経験レベルのコントリビューターを歓迎します。

## 質問がある場合

GitHub で Issue を作成するか、Discussion を開始してください。
